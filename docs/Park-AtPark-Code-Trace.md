## 1. Architecture Overview

The solution uses a **static facade + per-instance delegation** pattern:

| Layer | Class / File | Role |
|---|---|---|
| HTTP | `TelescopeController.cs` (ASCOM.Alpaca.Razor) | Receives ASCOM Alpaca REST requests |
| ASCOM Driver | `Telescope.cs` (GreenSwamp.Alpaca.Server) | ITelescopeV4 implementation |
| Instance facade | `MountInstance.cs` (GreenSwamp.Alpaca.MountControl) | Per-device state, SlewController, queues |
| Static facade | `SkyServer` (partial — multiple `SkyServer.*.cs` files) | Static wrapper delegating to `_defaultInstance` |
| Async state machine | `SlewController.cs` (GreenSwamp.Alpaca.MountControl) | 3-phase async slew (Setup → Movement → Completion) |
| Settings | `SkySettingsInstance.cs` (GreenSwamp.Alpaca.MountControl) | Per-instance settings with debounced JSON save |
| Persistence | `IVersionedSettingsService` (GreenSwamp.Alpaca.Settings) | Writes `appsettings.user.json` |

**Key architectural fact confirmed at `SkyServer.Core.cs` lines 69–71:**

```csharp
// _defaultInstance is a computed property — always slot 0 from registry

// _settings is a computed property — always _defaultInstance.Settings
private static SkySettingsInstance? _settings => _defaultInstance?.Settings;
```

`SkyServer._settings` and `_defaultInstance._settings` are **the same object reference** — both resolve through `MountInstanceRegistry.GetInstance(0).Settings`.

## 6. AtPark Two-Store Observation

`MountInstance` has **two separate stores** for park state:

### Store A — `private bool _atPark` (backing field)

Defined in `MountInstance.cs` backing fields region (approximately line 83):

```csharp
private bool _atPark;
```

Used exclusively inside `MountConnect()`:

```csharp
if (_atPark)
{
    // reset mount to default position at reconnect
}
```

### Store B — `SkySettingsInstance._atPark` (via property)

Accessed via `MountInstance.AtPark` property:

```csharp
public bool AtPark
{
    get => _settings.AtPark;
    set => _settings.AtPark = value;
}
```

Which reads/writes `SkySettingsInstance._atPark`.

### What is written where

- `SkyServer.CompletePark()` writes **Store B only** (`_settings.AtPark = true`).
- All external read paths (`Telescope.AtPark`, `CheckParked()`, the park guard in `Telescope.Park()`) read **Store B only**.
- `MountConnect()` reads **Store A only** (`_atPark` field).
- No code in the traced paths writes to **Store A** (`MountInstance._atPark` field) at run-time after construction. `ApplySettings()` writes to `SkySettingsInstance._atPark` (Store B), not to `MountInstance._atPark` (Store A).

## 8. Settings Persistence — Summary

`AtPark` is persisted as part of all-settings debounced save. The path is identical for all property changes:

```
Property setter
  └─ OnPropertyChanged()
       ├─ PropertyChanged event (Blazor UI binding)
       └─ QueueSave()
            └─ Task.Delay(2000ms) [restarts on each change]
                 └─ SaveAsync()
                      ├─ _settingsService.GetSettings()        ← loads current JSON model
                      ├─ maps 93+ fields (settings.AtPark = _atPark at line 2045)
                      └─ _settingsService.SaveSettingsAsync()  ← writes JSON file
```

File location: `%AppData%\GreenSwampAlpaca\{InformationalVersion}\appsettings.user.json`

On next application start, `SkySettingsInstance` constructor calls `ApplySettings()` which reads `settings.AtPark` and sets `SkySettingsInstance._atPark` (Store B). `MountInstance._atPark` (Store A) is not loaded from JSON by any path visible in this trace.
